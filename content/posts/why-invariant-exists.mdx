# Why Invariant Exists

AI can write code. That's settled.

What it cannot do — at least not reliably, not yet — is reason about *why* the code it writes works, *whether* it will scale, or *when* a fundamentally different approach is warranted. That requires understanding computation: Big O complexity, algorithmic design, system architecture.

The name **Invariant** comes from mathematics. An invariant is a property that holds true regardless of the transformations applied to a system. The thesis of this site is that certain principles of Computer Science are invariants — they remain true regardless of how AI transforms the industry.

This is that site. A laboratory for interactive algorithm exploration, a museum of the pioneers who built the field, and a ledger of essays on what still matters.

The audience is whoever taught themselves to code and suspects there is more to understand.
